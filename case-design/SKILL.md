# 需求到测试设计流水线

本技能以**七阶段流水线**为主线，将"需求 → 测试用例"的跨度拆解为可控的中间层，解决 AI 直接从需求生成用例时漏测严重的问题。核心思路：需求 → 对象 → 测试点 → 设计方法 → 场景 → 用例，每一层有明确输入输出，逐层收敛。

## 核心红线（贯穿全流程）

- **逐层推进，不跳层**：禁止从需求直接跳到测试用例（P1→P7），必须经过对象化（P3）和测试点（P4）中间层，否则漏测率不可控。
- **P1 是澄清循环不是审计**：需求评审采用多轮迭代的澄清模式，发现缺陷后必须转换为可执行的澄清问题，以 Quality Gate（G1-G5）全通过、成熟度达 L3 为完成标志，不能只列问题就结束。
- **P7 不再推理**：测试用例生成是机械翻译阶段，把 P6 场景按标准结构输出，不得在 P7 新增测试逻辑或臆测需求。
- **每层输出可追溯**：每个测试用例必须能回溯到测试点→需求对象→原始需求，断链即停。

## 流水线

按以下七阶段顺序推进。每个阶段有明确完成标志，未完成不进入下一阶段。

### ① P1 需求澄清与评审

将需求推进到可测试状态，多轮迭代。

- 读 [需求澄清与评审](p1-requirement-review.md)
- 输入：SRS / PRD / 用户故事
- 输出：需求澄清报告（问题列表带状态 + Delta Analysis + 成熟度评估）
- 完成标志：Quality Gate（G1-G5）全通过，成熟度达 L3（Testable）

### ② P2 需求解构

一次性把自然语言需求拆成结构化元素卡片（非循环）。

- 读 [需求解构](p2-requirement-analysis.md)
- 输入：P1 输出的成熟度 L3 需求（Quality Gate 全绿）
- 输出：结构化需求元素清单（每条需求一张 9 槽位卡片，有值或 N/A，不留空）
- 完成标志：每条需求卡片 9 类槽位填满；发现缺口不补齐，回溯 P1

### ③ P3 需求对象化

把 P2 的散装元素横向聚合成统一领域对象（一次性建模）。

- 读 [需求对象化](p3-requirement-object-modeling.md)
- 输入：P2 的结构化元素清单（元素须带归属标签）
- 输出：Requirement Object 模型（对象/属性/操作/关系/状态机五部分）
- 完成标志：模型文档生成，作为 P4 唯一输入；无法聚合则回溯 P2

### ④ P4 测试点提取

对领域模型地毯式扫描，穷举所有测试关注点（一次性）。

- 读 [测试点提取](p4-test-point-extraction.md)
- 输入：P3 的 Requirement Object 模型（属性类型/状态跃迁须完整）
- 输出：测试点清单 + 九大分类覆盖矩阵（命中记录，未命中标注 N/A 原因）
- 完成标志：清单 + 覆盖矩阵输出，无盲区；模型残缺则回溯 P3

### ⑤ P5 测试设计映射

为每个测试点匹配 ISTQB 设计方法，自动带出覆盖准则（一次性决策）。

- 读 [测试设计映射](p5-test-design-mapping.md)
- 输入：P4 的测试点清单（已分类、已标优先级）
- 输出：测试设计策略矩阵（主方法 + 覆盖准则 + 轻量验证结果）
- 完成标志：所有测试点完成匹配 + 验证扫描；High 优先级 Fail 则强制阻塞

### ⑥ P6 测试场景构建

把离散测试点按业务时序串成场景蓝图（一次性编排）。

- 读 [测试场景构建](p6-test-scenario-construction.md)
- 输入：P5 策略矩阵 + P3 领域模型（状态机/属性值域）
- 输出：场景蓝图集合（五类）+ 五类场景覆盖矩阵
- 完成标志：所有测试点归属场景 + 覆盖矩阵无空白；时序冲突则回溯 P4

### ⑦ P7 测试用例生成

把场景蓝图机械翻译为标准测试用例（一次性，不再推理）。

- 读 [测试用例生成](p7-test-case-generation.md)
- 输入：P6 场景蓝图集合 + P3 领域模型 + 操作手册/UI 原型
- 输出：标准测试用例集合（数据嵌入步骤，含图表/文件校验细化）
- 完成标志：所有场景翻译为用例，每条可回溯至场景ID和测试点ID

## 贯穿约束

以下约束不单独成阶段，内嵌于各阶段规则中：

| 约束 | 说明 | 体现于 |
|------|------|--------|
| 可追溯性 | 每层输出须能回溯上层，断链即停 | ①-⑦（每阶段输出均带来源标识） |
| 九大测试点分类 | 测试点提取与映射的分类基准 | ④定义、⑤引用、⑥覆盖矩阵纵轴 |
| ISTQB 设计方法集 | 测试设计映射的方法库 | ⑤（映射目标） |
| 五类场景路径 | 场景构建的骨架 | ⑥（Happy/Alternate/Exception/Permission/Recovery） |

## 认知架构

本流水线对应一个 Test Analyst Agent 的认知架构，七层各司其职：

| 层级 | 名称 | 作用 |
|------|------|------|
| L1 需求质量层 | Requirement Clarification & Review | 发现需求缺陷并推动澄清 |
| L2 需求理解层 | Requirement Analysis | 结构化解构 |
| L3 需求建模层 | Requirement Object Modeling | 统一领域对象 |
| L4 测试分析层 | Test Point Extraction | 抽取测试关注点 |
| L5 测试设计层 | Test Design Mapping | 绑定设计方法 |
| L6 场景构建层 | Test Scenario Construction | 组装业务场景 |
| L7 用例生成层 | Test Case Generation | 标准化输出 |
