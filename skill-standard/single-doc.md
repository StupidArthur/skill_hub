# 单文档 9 维规范

本章节定义单文档 skill 的编写规范。单文档 skill 的 9 个维度全部写在一个 SKILL.md 里，自包含。

## 适用条件

- 内容量中等，一个文件能讲完
- 内容连贯不宜拆分
- 单一职责的提示词技能

## 9 维度框架

SKILL.md 正文必须包含以下 9 个维度，按顺序组织：

### 1. Role (角色)
- **定义**：为 Agent 注入灵魂，明确专业身份。
- **撰写要求**：使用带有明确领域"定语"的专家头衔，例如 `资深 Web 前端自动化测试架构师`。

### 2. Profile (配置档案)
- **定义**：Prompt 的元数据字典与基础设定。
- **撰写要求**：必须按列表包含 `Author`, `Version`, `Language`, `Description`。

### 3. Background (背景说明)
- **定义**：任务的上下文与业务痛点。
- **撰写要求**：简明扼要说明为什么需要这个 Agent，解决什么具体问题。

### 4. Goals (核心目标)
- **定义**：Agent 的北极星指标。
- **撰写要求**：3-4 个简短的无序列表，指引工作方向。

### 5. Constraints / Rules (约束与规则)
- **定义**：防幻觉、防越界的绝对红线。
- **撰写要求**：用强硬祈使句（"必须"、"严禁"），明确三类约束（环境依赖、行为红线、异常熔断）。

### 6. Skills (核心技能) —— 强制必填
- **定义**：原子化能力。
- **撰写要求**：拆解为 3-5 个能力。每个能力**必须**包含：技能名称、实现逻辑、**依赖工具/接口**（如"通过 Chrome DevTools MCP 的 `click` 工具"）。

### 7. Workflows (工作流)
- **定义**：标准作业程序（SOP），即流程主线。
- **撰写要求**：有序列表，包含依赖识别与信息补全步骤。这是单文档 skill 的流程主线，必须清晰写出"先做什么、再做什么"。

### 8. Output Format (输出格式)
- **定义**：最终结果的呈现结构。
- **撰写要求**：使用缩进块（非反引号）表示示例。**红线**：此章节不得出现三个反引号包裹的代码块，避免与外层 Markdown 代码块嵌套冲突。

### 9. Initialization (初始化)
- **定义**：启动语。
- **撰写要求**：宣告就绪并明确告知用户输入格式。

## 编写流程

1. **接收需求**：读取用户提供的原始草案或业务需求。
2. **信息完整性评估**：评估是否覆盖核心节点（目标环境、操作步骤、预期结果）。
   - 若缺失超过 30% 关键信息：暂停输出，生成追问清单，严禁占位符。
   - 若信息基本完整：继续下一步。
3. **结构映射**：将原始信息映射到 9 维度框架。
4. **依赖与强约束识别**：提取"环境/工具强制依赖"，写入 Constraints 首部，映射到 Skills 的依赖标注。
5. **整合输出与自检**：组装完整 Markdown，扫描 Output Format 章节确保无嵌套反引号。

## 文件结构

单文档 skill 的目录结构：

```
skills/
└── <skill-name>/
    └── SKILL.md    ← 9 维内容全在这一个文件
```

单文档也是目录 + SKILL.md，只是目录里没有子文件。这样保证与目录级 skill 对 Agent 的加载机制完全一致。
