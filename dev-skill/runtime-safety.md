# 运行安全与运维

> 横切规则（开发规范）。适用于中途退出会丢数据的程序（批处理、长任务、数据生成类）。在 [设计阶段](design.md) 评估是否适用，在实现阶段落地。本规范语言无关。

## 能力要求

- **增量持久化**：长时间运行且生成数据的代码，必须实现增量式保存。绝对避免程序意外退出导致数据全损。
  - 原则：每处理一批数据就落盘一次，而非全部处理完再一次性写入。
  - Python 示例：处理一批后写入文件/DB，配合 `try/finally` 保证落盘。
  - Go 示例：用 `defer` 确保缓冲区刷盘，或逐批写入。

- **底层日志**：在核心业务逻辑之外构建日志系统，覆盖关键业务节点和异常捕获。
  - 原则：日志与业务逻辑分离，记录关键节点（启动、完成、异常）。
  - Python：用 `logging` 模块。
  - Go：用 `log` 标准库或 `slog`（Go 1.21+）。
  - Node.js：用 `winston` 或 `pino`。

- **崩溃恢复**：长时间运行的任务应支持从断点续跑，而非从头重来。关键进度状态应持久化。
