# 文档与知识管理

> 横切规则（产品规范）。指导各流程阶段的文档产出：[需求阶段](requirement.md) 产需求文档、[设计阶段](design.md) 产设计文档、[测试阶段](testing.md) 产测试记录。文档与代码必须同步，禁止滞后。

## 多步流程

每次完成重要功能的迭代后，若发现业务逻辑、架构设计或对外接口有变动，必须**主动检查并更新**以下文档（按项目实际情况，存在的才更新）：

1. 需求文档 — 记录功能需求与边界
2. 架构设计文档 — 记录模块划分与设计决策
3. 部署与运行手册 — 记录如何部署和运行
4. API 文档 — HTTP API 变更（前后端分离项目必须同步更新）

## 原则

- **文档与代码同一天**：文档与代码必须保持同步，禁止文档滞后于代码。
- **主动检查**：功能迭代后主动判断是否需要更新文档，而非等用户提醒。
- **变更对应**：API 接口变更必须同步到 API 文档，路由和参数定义严格对应。

## 文档位置

文档目录命名按项目惯例（如 `doc/`、`docs/`）。本规范不强制具体目录名，但要求文档与源码物理分离、结构清晰。
